# Mobile Chat UI Specification (Prototype)

## Core Screens
1. Welcome + consent screen
2. Profile onboarding chat
3. Today's plan card (diet + exercise)
4. Daily checkpoint chat
5. Progress dashboard summary
6. Safety escalation message panel

## Chat Flow Requirements
- Conversational intake for profile fields:
  - age, sex/gender, height, weight, activity level, goals
  - diet preference, allergies, cultural preferences
  - optional family history / DNA risk context
- Follow-up prompts for missing or invalid values
- Natural language + quick-reply buttons

## Plan Card Requirements
- Daily calories and protein targets
- Meal suggestions by time window
- Activity plan with duration/intensity
- Safety hints and adaptation notes

## Checkpoint Questions
- Did you complete your activity?
- How many steps/minutes?
- Did you follow meal plan?
- Energy (1-5), sleep quality (1-5)
- Any pain or unusual symptoms?

## Safety UX
When red flags are detected:
- Stop plan recommendations
- Display urgent-care guidance
- Provide emergency disclaimer
- Offer option to notify emergency contact (future enhancement)
